Інтеграція ChatGPT до додатків  Інтеграція ChatGPT до програми може здійснюватися різними способами залежно від передбачуваного використання. Ось деякі приклади:  •   Використання   API : Деякі служби пропонують API, який дозволяє отримати доступ до попередньо навченої моделі ChatGPT. Все, що вам потрібно зробити, це створити HTTP - запит, щоб отримати у відповідь JSON, містить вихідні дані моделі. Цю відповідь можна настроїти, змінивши параметри запиту. Ми вивчатимемо використання API у наступному розділі.  •   Використання бібліотеки : Існують такі бібліотеки, як як   Hugging   Face  Transformers , які дозволяють використовувати попередньо навчені моделі  ChatGPT   чи створювати власні моделі.  •   Створення власної моделі : Якщо у вас є великий набір даних, можна створити власну модель ChatGPT, навчивши її цих даних.
Після інтеграції ChatGPT можна використовувати в різних додатках, таких як чат - боти, ігри, віртуальні помічники тощо.

Він  надає простий використання API для взаємодії з моделями, дозволяючи генерувати текст, використовуючи задану вхідну інформацію.  •   OpenAI API : API OpenAI забезпечує доступ до моделей  GPT- 3 та   GPT- 2 безпосередньо на віддаленому сервері. Він  пропонує велику гнучкість з погляду конфігурації  моделі та параметрів введення/виводу. Ми будемо використовувати цей API у нашому прикладі.
•   Hugging Face API : API Hugging Face   також   надає  доступ до моделей ChatGPT, включаючи   GPT- 2 та   GPT- Neo. Всі ці API можуть використовуватися в JavaScript та пропонують різні функціональні можливості та рівні налаштування.

Функція   generateResponse()   викликається, коли користувач натискає кнопку   Send   (Надіслати). Вона отримує текст, введений користувачем, а потім надсилає запит POST в API OpenAI з відповідними параметрами, щоб отримати відповідь на запитання.  Відповідь відображається у блоці відповідей чат - бота.  Функція   reset () викликається, коли користувач натискає кнопку   Erase  (Скинути). Вона скидає вміст поля введення та блоку відповіді чат - бота.  Значення параметрів   model ,   max_tokens ,   temperature ,   top_p   та   n  передаються в API для налаштування відповіді чат - бота відповідно до потреб користувача. Значення   stop   не задано, оскільки API автоматично зупиняться після завершення відповіді (див. описання нижче).  API OpenAI також вимагає ключ API для автентифікації запиту.
Цей ключ включено до змінної заголовка. Ключ, що використовується тут, був отриманий безпосередньо з URL - адреси OpenAI  https://platform.openai.com/accountlapi - keys   . •   prompt : Це питання або запит, який слід використовувати для створення відповіді.  •   max tokens : Це максимальна кількість токенів, які можна згенерувати у відповіді.  •   temperature : Цей параметр керує креативністю генерованої відповіді (від 0 до 1). Більш високе значення цього параметра дозволяє творчіше реагувати, а нижче значення призводить до більш передбачуваної реакції.  •   top_p : Це ще один параметр (від 0 до 1), який керує креативністю відповіді, що генерується. Більше значення top_p також дозволяє давати більш креативні відповіді.  •   n : Це кількість відповідей, які потрібно згенерувати. Зазвичай це значення дорівнює 1 (n = 1).

Увімкнувши параметр   stop , можна проінструктувати модель припинити створення тексту, як тільки вона досягне цієї послідовності символів.  Це може бути корисним для налаштування відповідей, генеруються моделлю, відповідно до конкретних потребами програми або користувача.
